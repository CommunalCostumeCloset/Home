# 奇装异服共享平台：GitHub Pages + Cloudinary 版本

这个版本不需要 Node / Python 服务器，可以直接托管到 GitHub Pages。

## 已经填好的 Cloudinary 设置

```js
const CLOUD_NAME = 'vicummbs';
const UPLOAD_PRESET = 'returned_costumes_upload';
```

上传照片会使用 unsigned upload preset 上传到 Cloudinary。

## 重要安全提醒

不要把 API Secret 放进 GitHub Pages。API Secret 只能放在后端服务器里。你之前发过 secret，建议去 Cloudinary 后台 regenerate/rotate。

## 上传后的照片位置

代码会尽量把照片上传到：

```text
returned costumes/YYYY-MM-DD/
```

同时会加上 tag：

```text
returned_costume
return_YYYY_MM_DD
```

注意：如果 Cloudinary 的 unsigned preset 不允许前端覆盖 folder / asset_folder，照片仍然会进入你 preset 里设置的 `returned costumes` 文件夹，但管理员后台仍可按 tag 读取。

## 管理员后台如何显示所有照片

管理员后台使用 Cloudinary 的 client-side asset list by tag：

```text
https://res.cloudinary.com/vicummbs/image/list/returned_costume.json
```

你需要在 Cloudinary 里允许 Resource list：

1. 进入 Cloudinary Console
2. Settings
3. Security
4. 找到 Restricted image types
5. 取消勾选 Resource list / List
6. 保存

如果不做这一步，用户仍然可以上传照片，但是管理员后台无法从 GitHub Pages 公开读取照片列表。

## GitHub Pages 部署方法

1. 新建 GitHub repository
2. 上传这个文件夹里的 `index.html`
3. 进入 repository 的 Settings
4. Pages
5. Source 选择 `Deploy from a branch`
6. Branch 选择 `main` 和 `/root`
7. 保存

几分钟后 GitHub 会给你一个网址。

## 现实限制

这个版本适合 GitHub Pages 展示。它可以云端保存照片，但用户、衣物、预定记录仍主要存在浏览器 localStorage 里。也就是说：

- 同一台电脑刷新不会丢
- 换电脑可能看不到之前的预定状态
- Cloudinary 里照片是真实云端保存的

如果以后要做成真正全校可用的网站，需要再加一个云数据库，例如 Firebase Firestore、Supabase 或 Airtable。

## 新增：捐赠衣物必须上传照片

这一版已经取消衣物卡片里的卡通图标显示。捐赠衣物时，用户必须上传一张真实照片，否则无法提交。

捐赠照片会上传到 Cloudinary：

```text
donated costumes/YYYY-MM-DD/
```

同时会加上 tag：

```text
donated_costume
donation_YYYY_MM_DD
```

衣物目录会优先显示真实图片。如果是旧的示例衣物，因为没有真实照片，会显示“暂无真实照片”的占位图。

如果你已经开启了 Cloudinary Resource list，其他人打开网页时也会尝试读取 Cloudinary 里带 `donated_costume` tag 的捐赠衣物照片，并显示在衣物目录中。


## 新增：已取衣流程

这一版在“我的预定”中加入了 **已取衣** 按钮。

流程：

1. 用户先预定衣服。
2. 预定后，衣服状态是：`已预定，待取衣`。
3. 这个时候用户还可以点击 **取消预定**。
4. 用户拿到衣服后，点击 **已取衣**。
5. 点击后状态变成：`已取衣，待归还`。
6. 这时候不能再取消预定。
7. 这件衣服才会出现在“归还衣物”页面。
8. 唯一移除这条记录的方法是上传归还照片并确认归还。

这样可以避免用户只是预定了但还没真正拿走衣服时，系统就要求归还。

## 新增：管理员总表优化

这一版在管理员后台中加入了更清楚的状态总表：

## 新增：管理员确认归还流程

这一版加入了一个新的状态：

```text
处理中，等待管理员确认
```

新的完整流程：

1. 用户预定衣服：状态为 `已预定，待取衣`。
2. 用户点击 `已取衣`：状态变成 `已取衣，待归还`。
3. 用户上传归还照片：状态不会直接变成可借用，而是变成 `处理中，等待管理员确认`。
4. 管理员进入后台，在“衣物状态总表”里查看刚刚归还的人、上传时间和照片。
5. 管理员点击 `确认归还`。
6. 只有管理员确认后，衣服才会重新变成 `可借用`。

管理员表格排序现在是：

```text
已取衣，待归还
处理中，等待管理员确认
已预定，待取衣
可借用
```

“目前状态”一栏会根据当前状态显示不同内容：

- `已取衣，待归还`：显示“没有归还”、借用人、最晚归还时间。
- `处理中，等待管理员确认`：显示刚刚归还的人、上传时间、归还照片，并提供“确认归还”按钮。
- `已预定，待取衣`：显示“还在衣柜中”和预定归还时间，不再显示上一个人的归还信息。
- `可借用`：显示最近一次归还信息。

## 修复：衣物目录显示“处理中/待确认”

这一版修复了一个重要问题：用户上传归还照片后，衣服不会立刻回到可借用状态，而是在衣物目录中显示：

```text
处理中，等待管理员确认
```

管理员在后台点击“确认归还”之前，普通用户不能再次预定这件衣服。

### 为什么需要 Google Apps Script

GitHub Pages 只能运行前端网页，不能在每天固定时间自动检查逾期记录，也不能安全地保存 Gmail 密码。因此，本版本使用：

```text
GitHub Pages 网站
        ↓
Google Apps Script Web App
        ↓
Google Sheet 保存用户/预定记录
        ↓
每天检查逾期并从 Gmail 发提醒
```

### 第一步：用发件邮箱创建 Apps Script

1. 登录 `Communalcostumecloset@gmail.com`。
2. 打开 Google Apps Script，新建项目。
3. 把 `google-apps-script/Code.gs` 的全部内容粘贴到 Apps Script 的 `Code.gs`。
4. 在项目设置里确认时区。项目附带的 `appsscript.json` 使用 `America/New_York`；如果学校使用其他时区，请修改。
5. 在 Apps Script 编辑器顶部选择 `setup`，点击“运行”。
6. 第一次运行会要求授权 Gmail、Google Sheets 和触发器权限。
7. 运行成功后，打开“执行日志”。里面会显示自动创建的 Google Sheet 地址。

`setup()` 会自动：

- 创建 `Communal Costume Closet Data` Google Sheet；
- 创建 `Users` 和 `Reservations` 两个工作表；
- 创建每天约上午 9 点运行一次的逾期检查。

### 第二步：部署为 Web App

1. Apps Script 右上角点击 **Deploy → New deployment**。
2. 类型选择 **Web app**。
3. Execute as 选择 **Me**。
4. Who has access 选择 **Anyone**。
5. 点击 Deploy。
6. 复制生成的 Web App URL，通常以 `/exec` 结尾。

### 第三步：把 URL 填入网站

打开 `index.html`，搜索：

```js
const REMINDER_SERVICE_URL = '';
```

改成：

```js
const REMINDER_SERVICE_URL = '你的 Apps Script Web App URL';
```

然后把更新后的 `index.html` 上传到 GitHub Pages。

### 提醒规则

- `已预定，待取衣`：不发送逾期邮件，因为衣服仍在衣柜中。
- `已取衣，待归还`：超过最晚归还日期后发送提醒。
- `归还处理中`：不再发送提醒，因为用户已经上传归还照片。
- `管理员确认归还`：记录标记为完成，不再发送提醒。
- 同一件衣服同一天最多发送一次提醒；如果第二天仍未归还，会再次提醒。
